:::caution[For LavaMoat Developers]

This document is aimed at LavaMoat developers; the information herein may be of limited interest to end-users.

:::

LavaMoat is a monorepo organized into discrete packages. Below is a directory tree showing directories of interest, and how directories map to packages.

**Click a link to view detailed information about each directory or package.**

<FileTree>

- .config/ Common TypeScript, ESLint & other [configuration][config]
- .github/ [CI configuration][ci]
- packages/ Contains all npm workspaces
  - aa/ [**@lavamoat/aa**][lavamoat-aa] - Generates canonical package names
  - allow-scripts/ [**@lavamoat/allow-scripts**][lavamoat-allow-scripts] - Allow-list for on-install lifecycle scripts
  - browserify/ [**lavamoat-browserify**][lavamoat-browserify] - Browserify plugin
  - core/ [**lavamoat-core**][lavamoat-core] Core LavaMoat functionality
  - lavapack/ [**@lavamoat/lavapack**][lavamoat-lavapack] - Bundle packer for lavamoat-browserify
  - laverna/ [**@lavamoat/laverna**][lavamoat-laverna] - Tool to publish multiple workspaces in a monorepo
  - node/ [**lavamoat-node**][lavamoat-node] - Node.js adapter
  - preinstall-always-fail/ **@lavamoat/preinstall-always-fail** - Fails if preinstall lifecycle script is run; helper for [allow-scripts][lavamoat-allow-scripts]
  - tofu/ [**lavamoat-tofu**][lavamoat-tofu] - Generate LavaMoat policy from dependency graph
  - webpack/ [**@lavamoat/webpack**][lavamoat-webpack] - Webpack plugin
- scripts/ Custom [scripts][scripts]

</FileTree>
